websocket和http协议的区别

共同点:都是建立在TCP协议之上,通过TCP协议传输

 

不同点:

http: 单向请求服务,是客户端发起请求,服务端被动响应,一个request对应一个 response

websocket: 双向通信协议,握手阶段依靠http协议,握手后不再依托http协议,直接走tcp

 

HTTP三次握手四次挥手

三次握手: 1.C我要链接; 2.S我准备好了,你可连接; 3.C我收到你的同意命令了; 开始发送数据

四次握手:

1.C我要断开了,收到请回复;

2.S收到你的端口请求了,你等等我把数据发完

3.S数据发完了,可以断开,收到请回复;

4.C收到,断开;

实现原理

redis内部实现了一套自己的数据类型结构对象

这个对象的结构体, type 数据类型, encode 编码格式 ptr 实际数据对象地址

一种数据类型,存在多种编码格式,对应不同的底层结构体

encode 的编码类型有, 简单字符串, 整数类型, 压缩列表, 字典, 链表, 跳跃链表等
